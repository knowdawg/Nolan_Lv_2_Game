Quipsy

by: Nolan Kellerman

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I was interested in the old vector games such as Tempest and Star Wars so I wanted to make a game with lots of lines

2. What I Learned

I learned from player testing: - Spelling - Movement - Contrast - Speed What I learned about Java: - Fonts - Key release method - DrawLine method - Trigonometry

3. Architecture of My Game

My Game class runs the whole game because it creates my Paint Panel. the Paint Panel is responsible for creating and calling the update functions of all objects. The bullet object can collide with both enemies but not the player. The player can be moved by the arrow keys and can shoot the bullet objects. There are 2 enemies, Enemy and Nemesis. Enemy moves randomly and gets larger and larger the longer it lives. Nemesis is angry and targets the player directly by drawing an imaginary box around it and choosing a random point to move towards.

4. Technical Implementation

My favorite part of how the game works is how the green enemies get larger the longer they live. It was simple but it is a concept that really gives the player the urge to go fast. All I had to do was create a variable called fat and increase it by 1 every second. Whenever I drew the enemy or the collision box, I would simply add fat to it.

5. Most Challenging Part

The most challenging part of my project was the trigonometry. I didn't really know how to do trig, so I had to learn it.

6. Future Fixes and Updates

Some improvements I could make: - Constant running score - Global high score - Levels - Better menus - More enemies - A boss
